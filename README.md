# Очистка датасета с пропущенными значениями
Репозиторий с примерами и подходами к очистке датасетов, содержащих пропущенные значения (NaN/Null). Проект демонстрирует различные стратегии обработки данных данных с использованием Python и pandas.

## 📌 О проекте

Этот проект демонстрирует **профессиональный подход к очистке данных** на примере датасета розничных товаров. Исходные данные содержат **значительное количество пропусков** (до 63% в некоторых колонках), что делает их отличным кандидатом для применения различных стратегий обработки данных значений.

**Цель:** Превратить "грязный" датасет в чистый, структурированный и готовый к дальнейшему анализу или машинному обучению.

## 📊 Исходные данные

Датасет содержит информацию о товарах в розничной сети:
- **Category** — категория товара (A, B, C, D)
- **Price** — цена товара
- **Rating** — рейтинг товара (1-5)
- **Stock** — наличие на складе (In Stock / Out of Stock)
- **Discount** — размер скидки (%)

**Размер данных:** 4362 строки, 5 колонок

### Проблема: высокий уровень пропусков
| Колонка   | Пропуски (%) |
|-----------|--------------|
| Category  | 63.0%        |
| Price     | 4.0%         |
| Rating    | 47.0%        |
| Stock     | 31.0%        |
| Discount  | 9.0%         |

---

## 🔍 Процесс очистки: пошагово

### 1️⃣ Категория товара (`Category`)
**Проблема:** 63% пропусков — критический уровень, удалять строки нельзя (потеряем слишком много данных).  
**Решение:** Заполнение константой `'Unknown'`, чтобы сохранить все строки.

```python
df['Category'] = df['Category'].fillna('Unknown')
2️⃣ Цена (Price)
Проблема: Всего 4% пропусков.
Решение: Удаление строк с пропущенной ценой, так как цена — критически важный признак, и 4% — приемлемая потеря данных.

python
df = df.dropna(subset=['Price'])
3️⃣ Рейтинг (Rating) — продвинутая обработка
Проблема: 47% пропусков.
Решение: Комбинированный подход:

Создание флага Rating_Missing для отслеживания факта пропуска

Заполнение пропусков медианным рейтингом внутри каждой категории (логично, что товары одной категории имеют схожие оценки)

python
df['Rating_Missing'] = df['Rating'].isnull().astype(int)
df['Rating'] = df.groupby('Category')['Rating'].transform(
    lambda x: x.fillna(x.median())
)
4️⃣ Наличие на складе (Stock)
Проблема: 31% пропусков.
Решение: Аналогично рейтингу — создание флага Stock_Missing и заполнение 'Unknown'.

python
df['Stock_Missing'] = df['Stock'].isnull().astype(int)
df['Stock'] = df['Stock'].fillna('Unknown')
5️⃣ Скидка (Discount)
Проблема: 9% пропусков.
Решение: Логичное бизнес-решение — если скидка не указана, значит её нет. Заполняем 0 и создаём флаг.

python
df['Discount_Missing'] = df['Discount'].isnull().astype(int)
df['Discount'] = df['Discount'].fillna(0)

Данные готовы кEDA и построению моделей

💼 Ключевые бизнес-выводы
1. 63% товаров без категории — критическая проблема
Невозможно анализировать продажи по категориям и планировать закупки

Решение: Сделать поле "Категория" обязательным при добавлении товаров

2. Категория B — премиальный сегмент (цена на 13% выше средней)
Товары категории B (5410 ₽) — ядро высокомаржинальных продаж

Решение: Усилить маркетинг на эту категорию

3. Товары со скидкой имеют рейтинг ВЫШЕ (3.07 vs 2.99)
Скидки не означают "плохой товар", а наоборот — привлекают больше покупателей

Решение: Давать скидки на товары с высоким рейтингом для максимального эффекта

4. Цена и рейтинг не связаны (корреляция 0.016)
Дорогие товары не обязательно лучше — покупатели оценивают соотношение цена/качество

Решение: Не использовать "дорого = хорошо" как главный аргумент в маркетинге

5. 31% товаров без статуса наличия — прямые потери выручки
Система не знает, что можно продавать

Решение: Автоматизировать обновление статуса при изменении остатков

6. 47% товаров без отзывов — потеря доверия
Товары без рейтинга покупают реже

Решение: Запустить кампанию по сбору отзывов (бонусы, email-рассылки)
